# Core Behavioral Loops
Explain that this document defines the recurring behavioral patterns the app supports.
These loops are the foundation of motivation, retention, emotional regulation, and momentum.

---

# Morning Loop
Define the morning behavioral sequence:
- open app
- immediate orientation via Today
- see Focus Card
- complete one small task
- gain momentum
- reduce morning overwhelm

Specify:
- emotional goal
- cognitive load constraints
- interaction depth
- success criteria

## Sequence
1. **Open app**: Immediate orientation to Today screen
2. **See Focus Card**: Clear visual anchor showing current progress
3. **Complete one small task**: Quick win to build momentum
4. **Gain momentum**: Feel of accomplishment reduces overwhelm
5. **Start day**: Reduced anxiety, increased confidence

## Emotional Goal
- Reduce morning overwhelm and decision fatigue
- Build immediate sense of accomplishment
- Create positive momentum for the day
- Establish feeling of control and capability

## Cognitive Load Constraints
- Maximum 2 decisions required
- No complex navigation or setup
- Clear visual hierarchy with single focus point
- Predictable, repeatable pattern

## Interaction Depth
- 0-1 taps to complete primary action
- Inline interactions preferred
- No modal flows for morning loop
- Immediate visual feedback

## Success Criteria
- Complete at least one task within 60 seconds
- Feel reduced overwhelm after interaction
- Clear visual progress indication
- Return to app later in day

# Capture Loop
Define the rapid capture sequence:
- thought appears
- Quick Add
- minimal friction
- bottom sheet capture
- return to previous context

Specify:
- speed requirements
- friction rules
- return path rules
- ADHD-specific constraints

## Sequence
1. **Thought appears**: External trigger or internal reminder
2. **Quick Add**: Single tap to open capture
3. **Minimal friction**: Predictive defaults, minimal fields
4. **Bottom sheet capture**: Fast entry without context loss
5. **Return to previous**: Seamless return to original context

## Speed Requirements
- Capture must complete within 15 seconds
- Maximum 3 fields to complete
- No loading states or delays
- Immediate save and return

## Friction Rules
- No required fields except title/name
- Smart defaults for date, category, priority
- Keyboard optimized for content type
- Single-tap submission

## Return Path Rules
- Always return to previous screen
- Preserve scroll position and context
- No intermediate confirmation screens
- Visual confirmation of capture

## ADHD-Specific Constraints
- Capture before thought is lost (working memory)
- No complex decision-making during capture
- Immediate externalization of thought
- Minimal cognitive load during capture

# Recovery Loop
Define the re-entry sequence after missed days:
- no shame
- gentle reorientation
- simplified Today
- momentum restart
- optional catch-up suggestions

Specify:
- emotional safety rules
- cognitive load reduction
- progressive reactivation
- how the app avoids overwhelm

## Sequence
1. **No shame**: Welcome back message without guilt
2. **Gentle reorientation**: Simplified Today view
3. **Simplified Today**: Focus on current day only
4. **Momentum restart**: Single small action to rebuild
5. **Optional catch-up**: Gentle suggestions, not requirements

## Emotional Safety Rules
- No judgment for missed days
- Positive framing for return
- No overwhelming backlogs
- Celebrate return, not absence

## Cognitive Load Reduction
- Show only today's information
- Hide historical backlogs initially
- Single focus point for re-entry
- Minimal decisions required

## Progressive Reactivation
- Start with single small task
- Gradually reintroduce features
- Optional expansion of scope
- User-controlled pace of re-engagement

## Avoid Overwhelm
- No backlog notifications
- No "you missed X days" messaging
- Hide complex features initially
- Focus on present, not past

# Momentum Loop
Define the system that maintains ongoing engagement:
- streaks
- progress snapshots
- gentle nudges
- contextual reminders

Specify:
- reinforcement philosophy
- non-punitive design
- ADHD-safe motivation

## Streaks
- **Type**: Consistency-based, not perfection-based
- **Calculation**: Any engagement counts as streak maintenance
- **Recovery**: Easy streak recovery after breaks
- **Visualization**: Simple, clear progress indicators

## Progress Snapshots
- **Frequency**: Daily and weekly summaries
- **Content**: Accomplishments, not failures
- **Format**: Visual, scannable, positive
- **Timing**: Natural review points (end of day/week)

## Gentle Nudges
- **Type**: Contextual, not time-based
- **Content**: Helpful suggestions, not demands
- **Frequency**: Limited to avoid overwhelm
- **Control**: User-controlled notification preferences

## Contextual Reminders
- **Triggers**: Location, time, or pattern-based
- **Content**: Action-oriented, specific
- **Timing**: Just-in-time, not just-in-case
- **Respect**: User's current context and energy

## Reinforcement Philosophy
- **Positive reinforcement**: Focus on what works
- **Intrinsic motivation**: Build internal drive
- **Extrinsic support**: External tools for internal goals
- **Sustainable**: Long-term engagement patterns

## Non-Punitive Design
- **No shame**: No guilt-inducing messaging
- **No punishment**: No streak penalties or loss
- **No pressure**: No overwhelming demands
- **No judgment**: Accept user's current state

## ADHD-Safe Motivation
- **Immediate feedback**: Quick reinforcement
- **Clear progress**: Visual momentum indicators
- **Flexible engagement**: Multiple ways to succeed
- **Low barriers**: Easy to engage and disengage

# Loop Integrity Rules
Define rules that govern all loops:
- no loop may exceed 3 steps without payoff
- every loop must have a clear exit
- loops must reduce anxiety, not increase it
- loops must support low-energy states
- loops must be recoverable after interruption

## Maximum Steps Rule
- **3-step limit**: No loop exceeds 3 actions without payoff
- **Payoff frequency**: Reward at least every 3 steps
- **Complexity cap**: Break longer sequences into smaller loops
- **Exception**: Recovery loops may be longer for safety

## Clear Exit Rule
- **Always available**: Every loop has clear exit point
- **No traps**: User can always return to safe state
- **Predictable exits**: Exit points are consistent and discoverable
- **Graceful exit**: Exit doesn't lose progress or context

## Anxiety Reduction Rule
- **Calm design**: Loops must reduce, not increase anxiety
- **No pressure**: No overwhelming demands or deadlines
- **Predictable patterns**: Consistent, reliable interactions
- **Emotional safety**: Support user's emotional state

## Low-Energy Support Rule
- **Energy awareness**: Loops work with low energy levels
- **Minimal effort**: Reduce required cognitive/physical energy
- **Quick wins**: Provide immediate accomplishment
- **Flexible engagement**: Allow partial participation

## Recoverability Rule
- **Interruption recovery**: Loops recover from interruptions
- **Context preservation**: Maintain state across interruptions
- **Graceful degradation**: Function with partial engagement
- **Resumption support**: Easy to re-enter after breaks
